Задание: TRAFFIC ANALYSIS EXERCISE: LUMMA IN THE ROOM-AH
Источник: Malware-Traffic-Analysis
Условие задачи: 

Ваша задача: Будучи аналитиком в Центре операций безопасности (SOC), вы проверяете оповещения за последнюю неделю и находите срабатывание сигнатуры ET MALWARE Lumma Stealer Victim Fingerprinting Activity, которое было вызвано трафиком с адреса 153.92.1[.]49 через порт TCP 80. Оповещение сработало 27 января 2026 года в 23:05 UTC.

Используя эту информацию, вы извлекаете дамп сетевого трафика (pcap) с внутреннего IP-адреса, который инициировал это оповещение. На основе pcap вы пишете отчёт об инциденте, чтобы специалисты по реагированию могли отследить компьютер и связанного с ним пользователя.

Характеристики вашей среды:

    Диапазон сегмента LAN: 10.1.21[.]0/24 (от 10.1.21[.]0 до 10.1.21[.]255)
    Домен: win11office[.]com
    Название среды Active Directory (AD): WIN11OFFICE
    Контроллер домена Active Directory (AD): 10.1.21[.]2 — WIN-LU4L24X3UB7
    Шлюз сегмента LAN: 10.1.21[.]1
    Широковещательный адрес сегмента LAN: 10.1.21[.]255

Ваша задача: В рамках этого упражнения ответьте на следующие вопросы для вашего отчёта об инциденте:

    Каков IP-адрес заражённого Windows-клиента?
    Каков MAC-адрес заражённого Windows-клиента?
    Каково имя хоста (hostname) заражённого Windows-клиента?
    Каково имя учётной записи пользователя с заражённого Windows-клиента?
    Каково полное имя пользователя, связанное с этой учётной записью?
    Какой домен (или доменное имя) связан с 153.92.1[.]49, вызвавший оповещение для Lumma Stealer?

Решение:

1. Поиск IP-адреса заражённого клиента Я применил тот же подход, что и в предыдущем задании, чтобы отсеять шумовой трафик (SSDP, широковещательные запросы) и сосредоточиться на клиентских запросах к C2. Я использовал фильтр:

			(http.request or tls.handshake.type eq 1) and !(ssdp)

Этот фильтр показал все HTTP-запросы и TLS-рукопожатия, исключая SSDP. В результатах я заметил, что все подозрительные соединения с 153.92.1.49:80 исходят с одного внутреннего адреса:

				IP-адрес заражённого клиента: 10.1.21.58

2. Определение MAC-адреса. Зная IP-адрес (10.1.21.58), я кликнул по любому пакету с этим адресом и раскрыл детали канального уровня (Ethernet). В заголовке Ethernet я нашёл MAC-адрес отправителя:

				MAC-адрес заражённого клиента: 00:21:5d:c8:0e:f2

3. Получение hostname (имени компьютера). Вместо NBNS (как в прошлый раз) я решил поискать имя хоста в DHCP-трафике, так как при получении IP-адреса клиент передаёт своё имя в запросе. Я применил фильтр:
					
							dhcp

В одном из DHCP-пакетов я раскрыл структуру и нашёл поле Host Name Option, где было указано имя компьютера:
			
				Hostname заражённого клиента: DESKTOP-ES9F3ML

4. Определение имени учётной записи пользователя. Для поиска имени пользователя я использовал тот же метод, что и в прошлый раз – фильтрацию Kerberos-трафика. Я применил фильтр:

					kerberos && ip.addr eq 10.1.21.58

В одном из билетов Kerberos (TGT) я нашёл поле CNameString, которое содержало имя пользователя:

					Имя учётной записи: gwyatt

5. Установление полного имени пользователя. Чтобы найти полное имя пользователя, я воспользовался функцией поиска в Wireshark (Edit → Find Packet). В строке поиска я ввёл предполагаемое имя Wyatt (полученное на предыдущем шаге), выбрал поиск строки с учётом регистра и отметил опцию "Multiple occurrences" (множественные вхождения). Wireshark нашёл несколько вхождений в контексте протоколов SMB, LDAP или Kerberos. В одном из них было указано полное имя пользователя:

					Полное имя пользователя: Gabriel Wyatt

6. Определение вредоносного домена C2-сервера. Для идентификации домена, связанного с 153.92.1.49, я решил посмотреть TLS-рукопожатие (если оно есть) или DNS-запросы. Я применил фильтр для поиска TLS-рукопожатий с этим IP:
				
				tls.handshake.type eq 1 && ip.addr eq 153.92.1.49

В расшифрованном (или частично расшифрованном) TLS-рукопожатии в поле Server Name Indication (SNI) я нашёл доменное имя:

					Домен C2-сервера: whitepeper.su

Результаты расследования:

IP-адрес заражённого клиента	10.1.21.58
MAC-адрес			00:21:5d:c8:0e:f2
Hostname			DESKTOP-ES9F3ML
Имя учётной записи		gwyatt
Полное имя пользователя		Gabriel Wyatt
Домен С2-сервера		whitepeper.su

7. Заключение

Это упражнение помогло мне освоить различные методы поиска информации в pcap-файлах: использование DHCP для получения hostname, Kerberos для идентификации пользователя и TLS SNI для определения домена C2. Я закрепил навыки работы с фильтрами Wireshark и поиском по пакетам, что является важной частью работы аналитика SOC.
